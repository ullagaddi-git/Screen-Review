## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can run the app and see something functional — even if features are incomplete.
2. **Infrastructure before features.** Phase 0 sets up everything so feature work in later phases is fast and clean. Never skip foundation work to get to features faster.
3. **Magic moment first.** The terminal-error-to-AI-diagnosis flow must work by the end of Phase 2. Every architectural decision in Phases 0 and 1 is evaluated against enabling this.
4. **Test as you go.** Each task includes a verification step. Don't move to the next task until the current one is confirmed working. Save debugging for when things break, not at the end of a phase.
5. **Progressive enhancement.** Start with the simplest working version of each feature, then layer on polish. A working but rough auto-scroll is better than a perfect feature that blocks shipping.
6. **Review before proceeding.** After completing each phase, push a PR and let [CodeRabbit](https://coderabbit.ai) review it automatically. Do not start the next phase until the PR is reviewed and merged. This catches issues when they're cheap to fix.
7. **Local first, always.** Never write code that assumes network access for core functionality. Test every feature with the network disabled.

## Agent Session Guide

### How to Use This Roadmap with Your Coding Agent (Cursor)

1. **Start a phase:** Give your coding agent the phase prompt exactly as written above. The prompt tells the agent which reference sections to read — this keeps context lean.
2. **Read selectively:** Each phase lists only the PRD and vision doc sections relevant to that phase. The agent should read only those sections, not the entire documents.
3. **One task at a time:** The agent reads the first unchecked task, implements it, verifies it, marks it `- [x]`, then moves to the next. Don't let it work on multiple tasks simultaneously.
4. **One session = one phase (ideally):** Try to complete a full phase in a single Cursor session for best continuity. If you stop mid-phase, the agent can resume from the first unchecked task.
5. **Push a PR after every phase:** When a phase is complete, the agent should create a branch, commit all work, push, and open a PR. Let [CodeRabbit](https://coderabbit.ai) review it automatically before starting the next phase.
6. **Don't skip tasks:** Tasks are ordered intentionally. Skipping creates dependency issues that are painful to debug later.

**Starting a new phase:**

Then read only the Reference sections listed for that phase from `docs/prd.md` and `docs/product-vision.md`. Start working on the first unchecked task. After completing each task, update the checkbox to [x] in the roadmap file. Continue through all tasks in the phase."
